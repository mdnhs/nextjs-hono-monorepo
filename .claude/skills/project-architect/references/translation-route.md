# Translation — Route

Route-based i18n using `next-intl` with App Router routing. All routes move under `app/[locale]/`.
The root `app/layout.tsx` is a passthrough; the locale layout owns `<html>` and `<body>`.

### `src/lib/translation/request.ts`

```typescript
import { getRequestConfig } from 'next-intl/server';
import { hasLocale } from 'next-intl';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  const requested = await requestLocale;
  const locale = hasLocale(routing.locales, requested) ? requested : routing.defaultLocale;

  return {
    locale,
    messages: (await import(`./messages/${locale}.json`)).default,
  };
});
```

### App structure

All routes live under `app/[locale]/`. Replace the Phase 2 Step 3 folder skeleton with this:

```
src/app/
├── [locale]/
│   ├── layout.tsx              ← locale root: owns <html lang>, wraps providers
│   ├── (auth)/
│   │   └── layout.tsx
│   ├── (main)/
│   │   ├── (protected)/
│   │   │   ├── (dashboard_layout)/layout.tsx
│   │   │   └── (global_layout)/layout.tsx
│   │   └── (public)/
│   │       ├── unauthorized/page.tsx
│   │       └── maintenance/page.tsx
│   └── page.tsx
└── layout.tsx                  ← root: passthrough only (no html/body)
```

### `src/app/layout.tsx` — root passthrough

```typescript
import type { ReactNode } from 'react';

// The locale layout owns <html> and <body> so lang can be set per-locale.
// This root layout is required by Next.js but acts as a passthrough.
export default function RootLayout({ children }: { children: ReactNode }) {
  return children;
}
```

### `src/app/[locale]/layout.tsx`

**Do NOT write a hardcoded template for this file.** Instead, follow these steps:

1. **Read** the existing `src/app/layout.tsx` verbatim (the file generated by `create-next-app` + `shadcn init`,
   which already has the user's fonts, metadata, className values, and imports).

2. **Create** `src/app/[locale]/layout.tsx` as a direct copy of that file, with one path adjustment:
   - Change any `'./globals.css'` import to `'../globals.css'` (the locale layout is one level deeper).

3. **Apply** these route-translation-specific modifications to the copied file:

   a. Add imports (merge with existing imports, do not duplicate):

   ```typescript
   import { notFound } from 'next/navigation';
   import { hasLocale } from 'next-intl';
   import { setRequestLocale } from 'next-intl/server';
   import { routing } from '@/lib/translation/routing';
   import ProviderWrapper from '@/contexts/ProviderWrapper';
   ```

   b. Change the layout function signature to accept `params` and make it `async`:

   ```typescript
   export default async function LocaleLayout({
     children,
     params,
   }: {
     children: ReactNode;
     params: Promise<{ locale: string }>;
   });
   ```

   c. At the top of the function body, add:

   ```typescript
   const { locale } = await params;
   if (!hasLocale(routing.locales, locale)) notFound();
   setRequestLocale(locale);
   ```

   d. On the `<html>` element, replace any hardcoded `lang="en"` (or `lang` attribute) with `lang={locale}`.

   e. Replace the raw `{children}` inside `<body>` with `<ProviderWrapper>{children}</ProviderWrapper>`.

   f. Add the static params export (after any existing `metadata` export):

   ```typescript
   export function generateStaticParams() {
     return routing.locales.map((locale) => ({ locale }));
   }
   ```

   **Preserve everything else from the original** — font variables, metadata, className values,
   `suppressHydrationWarning`, and any other attributes or imports the user had.

### `src/contexts/ProviderWrapper.tsx` — updated by this feature

`NextIntlClientProvider` is added as the outermost wrapper inside `ProviderWrapper`.

```typescript
'use client';

import { NextIntlClientProvider } from 'next-intl';
import LoadingOverlayProvider from '@/contexts/LoadingOverlayProvider';
import { ReactNode } from 'react';

export default function ProviderWrapper({ children }: { children: ReactNode }) {
  return (
    <NextIntlClientProvider>
      <LoadingOverlayProvider>
        {children}
      </LoadingOverlayProvider>
    </NextIntlClientProvider>
  );
}
```

### `src/components/layout/locale-switcher.tsx`

```typescript
'use client';

import { usePathname, useRouter } from '@/lib/translation/navigation';
import { routing } from '@/lib/translation/routing';
import type { Locale } from '@/lib/translation/locale/constants';

const LOCALE_LABELS: Record<Locale, string> = {
  en: 'English',
  bn: 'বাংলা',
};

export function LocaleSwitcher({ currentLocale }: { currentLocale: Locale }) {
  const pathname = usePathname();
  const router = useRouter();

  function switchLocale(locale: Locale) {
    router.replace(pathname, { locale });
  }

  return (
    <div className='flex gap-2'>
      {routing.locales.map((locale) => (
        <button
          key={locale}
          onClick={() => switchLocale(locale as Locale)}
          aria-current={locale === currentLocale ? 'page' : undefined}
          className={locale === currentLocale ? 'font-bold' : 'opacity-60 hover:opacity-100'}
        >
          {LOCALE_LABELS[locale as Locale]}
        </button>
      ))}
    </div>
  );
}
```

## Usage patterns

### Server Components

```typescript
import { getTranslations } from 'next-intl/server';

export default async function Page() {
  const t = await getTranslations('common');
  return <button>{t('save')}</button>;
}
```

### Client Components (no prop drilling needed)

```typescript
'use client';
import { useTranslations } from 'next-intl';

export function MyClientForm() {
  const t = useTranslations('common');
  return <button>{t('save')}</button>;
}
```

### Navigation (locale-prefix auto-handled)

```typescript
import { Link, redirect, useRouter } from '@/lib/translation/navigation';
```

### Adding translations for a new feature

1. Add keys to `src/lib/translation/messages/en.json` and `src/lib/translation/messages/bn.json` under a feature namespace:
   ```json
   { "orders": { "title": "Orders", "createNew": "Create Order" } }
   ```
2. Access via `t('orders.title')` in server or `useTranslations('orders')` in client.
